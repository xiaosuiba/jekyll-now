生成自签名证书

如果您在公共网络上构建容器 linux 集群，推荐您为容器服务启用加密来避免流量探测以及中间人攻击。为了达到这个目的，您需要使用证书颁发机构（CA）、私钥和 CA 签发的证书。让我们一起使用 cfssl 来进行创建这些组件的整个流程。

注意： 此处我们将进行基本的流程。如果您希望配置高级安全选项，请参考官方 cfssl 文档。

下载 cfssl

CloudFlare 在 github 页面上分发 cfssl 源码，并在 cfssl 网站上分发二进制文件。

本文假设您在本地 x86_64 Linux 主机上运行 cfssl。

mkdir ~/bin
curl -s -L -o ~/bin/cfssl https://pkg.cfssl.org/R1.2/cfssl_linux-amd64
curl -s -L -o ~/bin/cfssljson https://pkg.cfssl.org/R1.2/cfssljson_linux-amd64
chmod +x ~/bin/{cfssl,cfssljson}
export PATH=$PATH:~/bin

初始化 CA

首先，我们需要保存默认 cfssl 选项，用于以后做替换：

mkdir ~/cfssl
cd ~/cfssl
cfssl print-defaults config > ca-config.json
cfssl print-defaults csr > ca-csr.json

在容器 Linux 中使用的证书类型

  • 客户端证书（client certificate） 用于从服务器验证客户端。例如 etcdctl、etcd proxy 或者 docker 客户端。
  • 服务端证书（server certificate） 用于服务器，客户端会通过它验证服务器的身份。例如 docker 服务器或者 kube-apiserver。
  • 同行证书（peer certificate） 用于 etcd 集群成员彼此之间进行双向通信。

配置 CA 选项

现在我们可以配置 ca-config.json 文件中的签名选项了。默认选项包含了以下预先配置的字段：

  • profiles：使用 X509 V3 扩展的 server auth（TLS Web Server Authentication） 默认值为 www，client auth(TLS Web Client Authentication) 默认值为 client。
  • expiry：默认值为 8760h（也即 365 天）。

规范起见，我们将 www 重命名为 server，创建同时使用 server auth 和 client auth 扩展的 peer 配置，并设置 expiry 为 43800h（5 年）：

{
    "signing": {
        "default": {
            "expiry": "43800h"
        },
        "profiles": {
            "server": {
                "expiry": "43800h",
                "usages": [
                    "signing",
                    "key encipherment",
                    "server auth"
                ]
            },
            "client": {
                "expiry": "43800h",
                "usages": [
                    "signing",
                    "key encipherment",
                    "client auth"
                ]
            },
            "peer": {
                "expiry": "43800h",
                "usages": [
                    "signing",
                    "key encipherment",
                    "server auth",
                    "client auth"
                ]
            }
        }
    }
}

{
    "CN": "My own CA",
    "key": {
        "algo": "rsa",
        "size": 2048
    },
    "names": [
        {
            "C": "US",
            "L": "CA",
            "O": "My Company Name",
            "ST": "San Francisco",
            "OU": "Org Unit 1",
            "OU": "Org Unit 2"
        }
    ]
}

然后使用配置的选项生成 CA：

cfssl gencert -initca ca-csr.json | cfssljson -bare ca -

您将得到以下文件：

ca-key.pem
ca.csr
ca.pem
  • 请你妥善保管 ca-key.pem 文件。这个密钥允许在您的 CA 内生成任意类型的证书。
  • *.csr 在我们的示例中没有使用。

生成服务端证书

cfssl print-defaults csr > server.json

服务端证书最重要的值是 Common Name (CN) 和 hosts。我们需要替换他们，例如：

...
    "CN": "coreos1",
    "hosts": [
        "192.168.122.68",
        "ext.example.com",
        "coreos1.local",
        "coreos1"
    ],
...

现在我们已经准备生成服务端证书和密钥：

cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=server server.json | cfssljson -bare server

或者不使用 CSR json 文件：

echo '{"CN":"coreos1","hosts":[""],"key":{"algo":"rsa","size":2048}}' | cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=server -hostname="192.168.122.68,ext.example.com,coreos1.local,coreos1" - | cfssljson -bare server

您将得到以下文件：

server-key.pem
server.csr
server.pem

生成同行证书

cfssl print-defaults csr > member1.json

替换 CN 和 hosts 值，例如：

...
    "CN": "member1",
    "hosts": [
        "192.168.122.101",
        "ext.example.com",
        "member1.local",
        "member1"
    ],
...

现在我们准备生成 member1 证书和密钥：

cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=peer member1.json | cfssljson -bare member1

或者不使用 CSR json 文件：

echo '{"CN":"member1","hosts":[""],"key":{"algo":"rsa","size":2048}}' | cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=peer -hostname="192.168.122.101,ext.example.com,member1.local,member1" - | cfssljson -bare member1

您将的得到以下文件：

member1-key.pem
member1.csr
member1.pem

为每个 etcd 成员的 hostname 重复这些步骤。

生成客户端证书

cfssl print-defaults csr > client.json

对于客户端证书，我们可以忽略 hosts 值，仅仅设置将 Common Name (CN) 值设置为 client：

...
    "CN": "client",
    "hosts": [""],
...

生成客户端证书

cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=client client.json | cfssljson -bare client

或者不使用 CSR json 文件：

echo '{"CN":"client","hosts":[""],"key":{"algo":"rsa","size":2048}}' | cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=client - | cfssljson -bare client

您将得到以下文件：

client-key.pem
client.csr
client.pem

TLDR

下载二进制

mkdir ~/bin
curl -s -L -o ~/bin/cfssl https://pkg.cfssl.org/R1.2/cfssl_linux-amd64
curl -s -L -o ~/bin/cfssljson https://pkg.cfssl.org/R1.2/cfssljson_linux-amd64
chmod +x ~/bin/{cfssl,cfssljson}
export PATH=$PATH:~/bin

创建目录保存证书：

mkdir ~/cfssl
cd ~/cfssl

生成 CA 证书

echo '{"CN":"CA","key":{"algo":"rsa","size":2048}}' | cfssl gencert -initca - | cfssljson -bare ca -
echo '{"signing":{"default":{"expiry":"43800h","usages":["signing","key encipherment","server auth","client auth"]}}}' > ca-config.json
export ADDRESS=192.168.122.68,ext1.example.com,coreos1.local,coreos1
export NAME=server
echo '{"CN":"'$NAME'","hosts":[""],"key":{"algo":"rsa","size":2048}}' | cfssl gencert -config=ca-config.json -ca=ca.pem -ca-key=ca-key.pem -hostname="$ADDRESS" - | cfssljson -bare $NAME
export ADDRESS=
export NAME=client
echo '{"CN":"'$NAME'","hosts":[""],"key":{"algo":"rsa","size":2048}}' | cfssl gencert -config=ca-config.json -ca=ca.pem -ca-key=ca-key.pem -hostname="$ADDRESS" - | cfssljson -bare $NAME

验证数据

openssl x509 -in ca.pem -text -noout
openssl x509 -in server.pem -text -noout
openssl x509 -in client.pem -text -noout

需要知道的事情

  • 不要将您的 ca-key.pem 放到容器 Linux 配置中，推荐将其保存在安全的位置。这个密钥允许创建任意数量的证书。
  • 妥善保管 key 文件。不要忘记设置合适的文件权限，例如 chmod 0600 server-key.pem。
  • 在这个 TLDR 示例中包含了使用 X509 V3 扩展的 server auth 和 client auth 证书，您可以将他们用于服务端与客户端认证。
  • 您也可以为通配符 * 地址生成密钥和证书。这些证书可以在任意机器上运行。这简化了证书创建但增加了安全风险。
